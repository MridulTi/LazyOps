# Roadmap

## v0.1 — Foundation

* CLI initialization
* Workflow discovery
* Manifest parsing
* Validation
* Workflow execution
* Interactive input prompts
* Basic documentation
* Unit tests

Goal:
Execute local workflows reliably.

---

## v0.2 — Developer Experience

* Search by name and tags
* Improved CLI output
* `list` command
* `describe` command
* Automatic documentation generation
* Better validation messages
* Configuration file support

Goal:
Make workflows easy to discover and understand.

---

## v0.3 — Sharing

* Install workflow packs
* Git-based workflow sources
* Version pinning
* Update notifications
* Local caching
* Export and import capabilities

Goal:
Enable sharing and reuse across teams.

---

## v0.4 — Ecosystem

* Runtime plugin interface
* Community-contributed runtimes
* Optional telemetry
* Execution history
* Workflow templates
* Workflow scaffolding

Goal:
Support a growing ecosystem without changing the core.

---

## v1.0 — Stable Platform

* Stable public APIs
* Backward-compatible workflow specification
* Optional web dashboard
* Rich filtering and search
* Team workflow libraries
* Cross-platform support
* Comprehensive documentation

Goal:
Become the standard way engineers organize and execute reusable automation workflows.

## Guiding Principles

Every release should improve one or more of:

* Simplicity
* Portability
* Discoverability
* Reusability
* Developer productivity

Features that compromise these principles should be rejected regardless of popularity.
